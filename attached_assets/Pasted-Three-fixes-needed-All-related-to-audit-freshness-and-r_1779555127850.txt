=== FIX 1: DASHBOARD SYNCS AFTER AGENT AUDIT ===

When agent runs live audit via run_audit tool,
save results to database and refresh dashboard.

In run_audit tool handler:
async function handleRunAuditTool(domain) {
  const result = await runFullAudit(domain)
  
  // Save to audits table
  await saveAuditToDatabase(result)
  
  // Clear cache so dashboard shows fresh data
  await db.delete(auditCacheTable)
    .where(eq(auditCacheTable.domain, domain))
  
  // Update monitored_brands last_checked
  if (userId) {
    await db.update(monitoredBrandsTable)
      .set({ lastChecked: new Date() })
      .where(eq(
        monitoredBrandsTable.domain, domain))
  }
  
  return result
}

After tool completes emit refresh event:
window.dispatchEvent(
  new CustomEvent('audit-updated', {
    detail: { domain: domain }
  })

=== FIX 3: RECOMMENDATIONS MUST REFLECT 
CURRENT STATE — NOT CACHED STATE ===

This is the most important fix.

CURRENT BROKEN BEHAVIOR:
1. User runs audit → robots.txt scores 0
2. Recommendations say "fix robots.txt"
3. User fixes robots.txt on their site
4. User reruns audit → robots.txt now 100
5. Recommendations STILL say "fix robots.txt"
6. This is wrong — issue is already fixed

The problem: recommendations are generated
ONCE and cached. They never regenerate
based on what has actually been fixed.

THREE PARTS TO FIX:

PART A — Regenerate recommendations 
on every fresh audit

Recommendations must be generated fresh
every time a new audit runs.

Never use cached recommendations.
Never show recommendations from a
previous audit run.

In audit pipeline:
// Always regenerate — never cache recs
const recommendations = 
  await generateRecommendations(
    brandName,
    domain,
    category,
    market,
    FRESH_scores,  // use current scores
    FRESH_technical_checks,  // current checks
    keywords
  )

// Save fresh recommendations
await saveRecommendations(
  auditId, recommendations)

PART B — Recommendations must check 
current technical state before suggesting

Before generating recommendations,
check current state of each technical item:

const currentState = {
  robotsFixed: technicalChecks.robots.score > 50,
  llmsExists: technicalChecks.llms.score > 50,
  schemaAdded: technicalChecks.schema.score > 50,
  contentGood: technicalChecks.content.score > 70,
  entityGood: technicalChecks.entity.score > 50
}

Pass currentState to Claude recommendation prompt:

"Generate 5 fix recommendations.
IMPORTANT — do NOT recommend things 
that are already done:

Current technical state:
robots.txt AI crawlers: ${currentState.robotsFixed ? 
  'ALREADY FIXED — do not recommend' : 
  'NEEDS FIX — recommend this'}
  
llms.txt file: ${currentState.llmsExists ? 
  'ALREADY EXISTS — do not recommend' : 
  'MISSING — recommend this'}
  
Schema markup: ${currentState.schemaAdded ? 
  'ALREADY ADDED — do not recommend' : 
  'MISSING — recommend this'}

Only recommend things that are 
actually missing or broken right now.
Focus recommendations on what will 
move the score from its CURRENT value
of ${currentScore}/100."

PART C — Fix Actions tab shows 
completed items differently

When a technical check passes (score > 70):
Mark that task in Fix Actions as completed:

[I] robots.txt ← if score = 100
Show as: 
  ✓ completed
  strikethrough text

Do NOT show as an active task

For citation tasks (cannot auto-verify):
