Add a new "Content Improvements" tab 
to the paid dashboard.

This is inspired by Rankscale's 
page audit content improvement feature.

It is the most valuable feature 
we can add right now.

=== WHAT IT DOES ===

1. Scrape user's homepage content
2. Identify specific sections of text
3. For each section show:
   - Current content (what they have)
   - Improved version (AI-rewritten)
   - Why it improves AI visibility
4. User copies improved version
   and replaces it on their site

=== SECTIONS TO IMPROVE ===

Analyze and improve these sections:

1. Hero Headline
   Current: main H1 text
   Improve: add question format,
   include target keywords,
   make AI-extractable

2. Hero Subheadline  
   Current: subtitle/description text
   Improve: add specifics, numbers,
   named conditions or use cases

3. Main Value Proposition
   Current: "what we do" paragraph
   Improve: more descriptive,
   conversational, citable

4. Feature/Benefit sections
   (one card per major benefit)
   Current: each benefit text
   Improve: expand with specifics,
   add "why it works" reasoning

5. Social Proof section
   Current: testimonials or stats
   Improve: make more specific,
   add data points AI can cite

6. CTA Section
   Current: call to action text
   Improve: add urgency + specifics

For health apps specifically:
Add improvements for:
- Condition-specific sections
  (diabetes, PCOS, thyroid etc)
- Comparison section
  (before/after using product)
- How it works section

=== AI GENERATION ===

For each section call Claude API:

Prompt:
"You are a GEO optimization expert.
Improve this website content section 
to be more visible in ChatGPT, 
Gemini and Perplexity.

Brand: {brand_name}
Category: {category}
Market: {market}
Target audience: {audience}
Top keywords: {keywords}

Current content:
{scraped_section_text}

Rules for improved version:
1. Start with a question format 
   when possible
2. Include specific numbers, 
   ingredients, data points
3. Use conversational natural language
4. Include regional specifics 
   (for India: mention Indian food names)
5. Make it AI-quotable in 1-2 sentences
6. Keep roughly same length as original
7. Do not change the meaning

Also explain in one sentence 
why this specific change improves 
AI search visibility.

Return JSON:
{
  'improved': 'improved text here',
  'reason': 'one sentence explanation'
}"

=== SECTION DETECTION ===

Parse homepage HTML to detect sections:

Look for:
- h1, h2, h3 tags
- p tags with 50+ words
- sections with class/id containing:
  hero, feature, benefit, about,
  testimonial, cta, comparison

Group nearby elements into sections.
Name each section by its H2/H3 heading
or by content type if no heading.
