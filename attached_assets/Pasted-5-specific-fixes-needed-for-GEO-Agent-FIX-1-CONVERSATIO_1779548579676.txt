5 specific fixes needed for GEO Agent.

=== FIX 1: CONVERSATION TONE ===

The agent writes like a robot using 
**markdown bold** and dumps walls of text.

Update the system prompt with these rules:

WRITING STYLE RULES:
1. Never use **bold** markdown in responses
2. Never use ## headers in responses
3. Never start sections with bold labels
   like **Tweet 1 (Problem-aware)**
4. Write in natural flowing paragraphs
5. Maximum 3 paragraphs per response
   unless writing actual content
6. When listing things use simple
   numbered lists: 1. 2. 3.
   Not bold headers
7. Sound like a smart colleague
   not a consultant writing a report
8. End every response with one clear
   next action question
   e.g. "Want me to write that article 
   for you right now?"

BAD example:
**Your GEO IQ Score of 0 means...**
**The three specific reasons:**
**1. No content depth**

GOOD example:
Your score is at zero because AI systems
have nothing to cite about MealCoreAI yet.
There is no published content they can
reference when someone asks about 
Indian diabetic meal planning.

The fastest fix is one well-written article.
Want me to draft it now?

Apply this to every response going forward.

=== FIX 2: SCORE SYNC ===

The agent shows technical score as 20/100
but the actual audit shows 66/100.

This happens because agent uses 
stale cached audit data.

Fix: When loading brand context for agent,
always fetch the LATEST audit from database:

SELECT * FROM audits
WHERE domain = $1
ORDER BY created_at DESC
LIMIT 1

Also fetch latest technical audit:
SELECT * FROM technical_audits
WHERE domain = $1
ORDER BY created_at DESC
LIMIT 1

Pass LATEST scores to system prompt.
Never use hardcoded or cached scores.

Also add to system prompt:
"The latest technical audit shows:
robots.txt: {latest_robots_status}
llms.txt: {latest_llms_status}
Schema: {latest_schema_status}
Content: {latest_content_status}
Entity: {latest_entity_status}
Technical total: {latest_technical_score}/100

Note: The user may have fixed issues
since the last AI visibility scan.
Always reference the most recent 
technical data available."

=== FIX 3: TWEET CARD PARSING ===

Tweet cards are parsing intro text
as tweet content.

Currently showing:
Card 1: "Assuming they are for your 
actual users, here are 3 tweets:"
(this is the intro not a tweet)

Fix the tweet parsing logic:

The Claude response format for tweets is:
---
**Tweet 1 (angle)**
[actual tweet text]

---
**Tweet 2 (angle)**
[actual tweet text]

Parser should:
1. Split on "---" separator
2. For each section:
   - Skip if it contains "here are" or
     "assuming" or "before I write"
   - Extract text AFTER the **Tweet X** line
   - That text is the actual tweet
3. Only render cards for actual tweet text

Also update the system prompt to format
tweets more cleanly:

"When writing tweets always format 
exactly like this with no variation:

TWEET 1 [angle name]
[tweet text here - nothing else]

TWEET 2 [angle name]  
[tweet text here - nothing else]

TWEET 3 [angle name]
[tweet text here - nothing else]

No intro text before TWEET 1.
No explanatory text between tweets.
No questions after tweets.
Just the three tweets in this format."

Keyword Opportunities table shows:
"Keyword data will appear after your 
next audit with DataForSEO credentials"

But DataForSEO is already configured.

Fix: The keyword table component is
checking for DataForSEO credentials
in the wrong place.

Update the keyword data loading:

1. Check keyword_cache table first:
   SELECT keywords FROM keyword_cache
   WHERE domain = $1
   AND expires_at > NOW()
   
2. If found: use cached keywords
   for the keyword opportunities table

3. If not found: call DataForSEO API
   and populate the table

4. Pass keywords to agent system prompt:
   "DataForSEO keywords for this brand:
   {keywords.map(k => 
     `- "${k.keyword}" (${k.volume}/mo)`
   ).join('\n')}"

The keyword table should show real data
immediately — not wait for next audit.
