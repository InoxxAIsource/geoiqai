=== STEP 3: BACKEND PAYMENT ROUTES ===

Create/update these API routes:

--- Route 1: Create subscription ---
POST /api/payment/create-subscription

Request body:
{
  "plan": "starter" or "agency",
  "email": "user@example.com",
  "domain": "theirdomain.com"
}

Handler:
1. Get plan_id from env based on plan type
2. Create Razorpay customer:
   {
     "name": email.split('@')[0],
     "email": email,
     "notify": { "email": true }
   }
3. Create subscription:
   {
     "plan_id": plan_id,
     "customer_notify": 1,
     "quantity": 1,
     "total_count": 12,
     "notes": {
       "email": email,
       "domain": domain,
       "plan": plan
     }
   }
4. Return:
   {
     "subscription_id": sub.id,
     "razorpay_key": RAZORPAY_KEY_ID,
     "customer_id": customer.id,
     "amount": plan_amount,
     "plan_name": plan_name
   }

--- Route 2: Verify payment ---
POST /api/payment/verify

Request body:
{
  "razorpay_payment_id": "pay_...",
  "razorpay_subscription_id": "sub_...",
  "razorpay_signature": "...",
  "email": "user@example.com",
  "domain": "theirdomain.com",
  "plan": "starter"
}

Handler:
1. Verify signature:
   const body = payment_id + "|" + subscription_id
   const expected = crypto
     .createHmac('sha256', RAZORPAY_KEY_SECRET)
     .update(body)
     .digest('hex')
   if expected !== razorpay_signature:
     return 400 "Invalid signature"

2. Create user in database:
   INSERT INTO users (
     email,
     razorpay_subscription_id,
     razorpay_customer_id,
     plan,
     created_at
   ) VALUES (...)
   ON CONFLICT (email) DO UPDATE
   SET plan = excluded.plan,
       razorpay_subscription_id = 
         excluded.razorpay_subscription_id

3. Generate magic login token:
   token = crypto.randomBytes(32)
             .toString('hex')
   Save to magic_tokens table:
   INSERT INTO magic_tokens (
     user_id, token, 
     expires_at = NOW() + 15 minutes
   )

4. Send welcome email via Resend:
   Subject: "Welcome to GeoIQ — 
             click to access dashboard"
   Body: include magic link:
   https://geoiqai.com/auth/magic
     ?token={token}
   Link expires in 15 minutes text
   "Request new link" instruction

5. Link any existing audit history:
   UPDATE email_subscribers 
   SET user_id = new_user.id
   WHERE email = email

6. Return:
   { 
     "success": true,
     "message": "Check your email 
       for dashboard access link"
   }

--- Route 3: Razorpay webhook ---
POST /api/payment/webhook

Handler:
1. Verify webhook signature:
   const body = req.rawBody
   const signature = req.headers
     ['x-razorpay-signature']
   const expected = crypto
     .createHmac('sha256', 
       RAZORPAY_WEBHOOK_SECRET)
     .update(body)
     .digest('hex')
   if expected !== signature:
     return 400

2. Handle events:

subscription.activated:
  → Set user.plan = 'starter' or 'agency'
  → Set user.subscription_status = 'active'
  → Log activation

subscription.cancelled:
  → Set user.plan = 'free'
  → Set user.subscription_status = 'cancelled'
  → Send cancellation email

subscription.charged:
  → Log successful payment
  → Send receipt email

payment.failed:
  → Send payment failed email
  → Keep plan active for 3 days grace

3. Always return 200 to Razorpay
   (even on errors — prevents retries)

--- Route 4: Magic link login ---
GET /api/auth/magic?token=xxx

Handler:
1. Find token in magic_tokens table
2. Check not expired 
   (expires_at > NOW())
3. Check not already used 
   (used_at IS NULL)
4. If valid:
   → Mark token used: 
     UPDATE magic_tokens 
     SET used_at = NOW()
   → Create session:
     req.session.userId = user.id
     req.session.email = user.email
     req.session.plan = user.plan
   → Redirect to /dashboard
5. If invalid/expired:
   → Redirect to /auth/login
     with error message

--- Route 5: Request magic link ---
POST /api/auth/request-magic-link

Request body: { "email": "..." }

Handler:
1. Find user by email in users table
2. If not found:
   Return: {
     "error": "No paid account found 
     for this email. 
     Check your email or upgrade at 
     geoiqai.com/pricing"
   }
3. If found and plan is active:
   → Generate new token
   → Send magic link email
   → Return: { 
     "success": true,
     "message": "Check your email 
       for login link"
     }

--- Route 6: Billing portal ---
GET /api/payment/portal

Handler (requires paid session):
1. Get user's razorpay_customer_id
2. Redirect to Razorpay customer portal
   Or show subscription management page

--- Route 7: Logout ---
POST /api/auth/logout
→ Destroy session
→ Redirect to homepage

=== STEP 6: DATABASE TABLES ===

magic_tokens table:
CREATE TABLE IF NOT EXISTS magic_tokens (
  id UUID PRIMARY KEY 
    DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  token TEXT UNIQUE NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  expires_at TIMESTAMPTZ DEFAULT 
    NOW() + INTERVAL '15 minutes',
  used_at TIMESTAMPTZ
);
